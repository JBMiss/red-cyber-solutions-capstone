# Red Cyber Solutions — Capstone Security Program Design

> A four-part security program design for a fictional cybersecurity consulting firm. Covers risk analysis, SOC modernization, ethical hacking program design, and tooling selection.

**Author:** Jesse Missaghian
**Program:** B.S. in Cybersecurity, University of Phoenix
**Courses:** CYB/490 (Capstone Design) • CYB/492 (Capstone Implementation)
**Timeline:** August – October 2024

---

## What this project is

Red Cyber Solutions is a **fictional** mid-sized cybersecurity consulting firm I designed across two capstone courses to serve as a recurring client. Every deliverable in this repo treats Red Cyber Solutions as the client and answers a realistic question their leadership team might ask:

> *"We have gaps in our security posture. What do we do about it, and how do we justify the spend?"*

Rather than a collection of disconnected assignments, the four deliverables below form a coherent program design — the kind of work product a security consultant or analyst would produce for an actual client engagement.

## Why this matters for a security role

This project demonstrates three things that hands-on lab work alone doesn't:

1. **Strategic thinking** — connecting technical choices to business objectives, budgets, and regulatory requirements
2. **Written communication** — every deliverable is written for a specific audience (executives, IT security team, senior leadership) in that audience's language
3. **Program design, not just task execution** — scoping, resourcing, justifying, and sequencing security work across a multi-quarter horizon

## The four deliverables

| # | Deliverable | Question it answers | Audience |
|---|---|---|---|
| [01](./01-risk-analysis) | **Risk Analysis Report** | How should we prepare for DDoS and ransomware attacks? | Security leadership |
| [02](./02-soc-migration) | **SOC Migration Decision** | Should we move our on-prem SOC to the cloud? | Executive management |
| [03](./03-ethical-hacking-program) | **Ethical Hacking Program Proposal** | How do we build a proactive offensive security capability? | Executive management + IT Security |
| [04](./04-language-selection) | **Tooling & Language Selection** | Python, Ruby, or both — which fits our ethical hacking program? | Senior leadership |

Each folder contains the full deliverable and a short brief explaining the context and what the work demonstrates.

## Key artifacts at a glance

- **$182,691** — detailed budget estimate for the ethical hacking program, broken down by personnel, tools, training, and contingency
- **8-phase** SOC-to-cloud migration plan covering assessment through compliance audit
- **20-week** program timeline with defined testing, analysis, remediation, and follow-up windows
- **14-slide executive presentation** comparing Python vs. Ruby for offensive security work
- Stakeholder analysis, legal/regulatory mapping (PCI DSS, HIPAA, GDPR, CFAA, CA Penal Code §502)

## How to read this repo

**If you have 5 minutes:** read this README and skim [`01-risk-analysis`](./01-risk-analysis) — it's the shortest and sets up the scenario.

**If you have 15 minutes:** add [`03-ethical-hacking-program`](./03-ethical-hacking-program). This is the deliverable with the most strategic depth — budget, stakeholders, methodology, timeline.

**If you have 30 minutes:** read all four in order. They build on each other.
